# Documentation Modes

The docs skill operates in one of four modes per document, determined during the
audit step.

## Ownership Tracking

Every document generated by this skill MUST include a marker as the first line:

```html
<!-- generated-by: docs-skill -->
```

This marker determines how the skill interacts with the file:

- **Marker present** → skill-owned document → eligible for `update` or `fix`
  mode
- **Marker absent** → hand-written document → eligible for `supplement` mode
  only

## Mode Selection

During the audit step (Step 2), determine the mode for each document:

| Condition | Mode |
|-|-|
| File does not exist | `create` |
| File exists with `<!-- generated-by: docs-skill -->` marker and is outdated | `update` |
| File exists without marker (hand-written) and is missing sections | `supplement` |
| File was flagged by verification with specific failing claims | `fix` |

## Create Mode

Write the document from scratch.

1. Select the matching reference template for the doc type
2. Explore the codebase using the Content Discovery section of the template
3. Write the complete document following the template's Required Sections
4. Include the ownership marker as the very first line
5. Place `<!-- VERIFY: {claim} -->` markers on infrastructure claims that cannot
   be verified from repo contents (URLs, server configs, external service
   details)

## Update Mode

Revise an existing skill-generated document.

1. Read the existing document
2. Compare each section against current codebase state
3. Identify sections that are inaccurate or missing vs. the reference template
4. Rewrite only inaccurate or missing sections — preserve accurate content
5. Ensure the ownership marker remains on the first line
6. Re-verify any `<!-- VERIFY: ... -->` markers — remove if now verifiable

**Key rule:** Update mode is not a rewrite. If 80% of the doc is accurate, only
touch the 20% that isn't.

## Supplement Mode

Append missing sections to a hand-written document. **Never modify existing
content.**

1. Read the existing document
2. Extract all `## ` headings from the file
3. Compare against the Required Sections from the matching reference template
4. For each section that exists in the template but is absent from the file:
   a. Explore the codebase to gather accurate facts
   b. Generate the section content following the template
5. Append all missing sections to the end of the file
6. Do NOT add the ownership marker — the file remains user-owned

**Critical:** Supplement mode must never modify, reorder, or rephrase any
existing line. Only append new `##` sections that are completely absent.

## Fix Mode

Correct specific failing claims identified by the verification agent. Only
modify the lines listed in the failures — nothing else.

1. Receive the failure list from the verification agent (line, claim, expected,
   actual)
2. For each failure:
   a. Locate the line in the document
   b. Search the codebase to find the correct value
   c. Replace only the incorrect claim with the verified-correct value
   d. If the correct value cannot be determined, replace with a
      `<!-- VERIFY: {claim} -->` marker
3. Ensure the ownership marker remains on the first line

**Critical:** Fix mode must correct only the listed failures. Do not modify,
reorder, rephrase, or "improve" any other content. The goal is surgical
precision.
